## 🎯 快速部署步驟 (Streamlit Cloud - 免費版)

### 步驟 1：準備 GitHub 儲存庫

1. 註冊 GitHub 帳號 (免費)：https://github.com
2. 建立新 Repository，名稱建議 `stock-god-battle`
3. 上傳以下 3 個檔案：
   - `stock_god_battle_v2.py` (主程式 - 重新命名為 `app.py` 較方便)
   - `requirements.txt` (套件清單)
   - `.streamlit/config.toml` (設定檔)

**檔案結構：**
```
stock-god-battle/
├── app.py              # 主程式 (由 stock_god_battle_v2.py 更名)
├── requirements.txt    # Python 套件
└── .streamlit/
    └── config.toml     # Streamlit 設定
```

### 步驟 2：部署到 Streamlit Cloud

1. 前往 https://streamlit.io/cloud
2. 用 GitHub 帳號登入
3. 點擊 "New app"
4. 選擇你的 Repository：`yourname/stock-god-battle`
5. 主檔案路徑填寫：`app.py`
6. 點擊 "Deploy"
7. 等待 2-3 分鐘部署完成
8. 獲得專屬網址：`https://stock-god-battle-xxx.streamlit.app`

### 步驟 3：手機操作

1. 將上述網址加入手機主畫面（Chrome/Safari 分享按鈕 → 加入主畫面）
2. 像使用 App 一樣直接開啟
3. 支援觸控操作、響應式介面

## ⚙️ 功能使用說明

### 角色設定 (3位股神)

每位股神可獨立設定：

| 設定項目 | 說明 |
|---------|------|
| **策略模板** | 短線MA交叉 / KD極值逆勢 / RSI超買超賣 / MACD趨勢 / 多指標綜合 |
| **回測月份** | 1-12個月歷史數據 |
| **初始資金** | 10萬 - 1000萬 |
| **啟用指標** | MA / KD / RSI / MACD / 布林通道 |
| **條件數** | 需滿足幾個指標條件才交易 (避免雜訊) |
| **進階參數** | 各指標的週期、閾值微調 |

### 對決戰場

- **即時顯示**：3位股神的資金、報酬率、勝率、交易次數
- **走勢圖表**：資金曲線比較、報酬率對比 (Plotly 互動圖表)
- **交易記錄**：每筆買賣的價格、損益、原因
- **自動排名**：依總報酬率即時排序

### 控制按鈕

- **開始對決**：所有股神開始依序執行交易
- **暫停**：停止模擬
- **重置角色**：個別或全部重置資金和進度

## 🔧 技術規格

### 數據來源
- **TWSE 台灣證交所**：`https://www.twse.com.tw/exchangeReport/STOCK_DAY`
- **免費公開 API**：無需 API Key，無用量限制（合理範圍內）
- **更新頻率**：日終收盤數據（一天一次）

### 交易規則
- **最小單位**：台股 1000 股 / 手
- **交易時機**：依策略訊號隔日執行（收盤價模擬）
- **破產條件**：資金不足以購買 1000 股且無持倉
- **手續費**：簡化模型未計入（可自行擴充）

### Streamlit Cloud 免費版限制 [^61^]
- **記憶體**：1GB RAM
- **儲存空間**：無持久化儲存（session state 重啟後重置）
- **睡眠機制**：30分鐘無訪問會進入睡眠，下次訪問喚醒
- **並發用戶**：適合個人使用或小團隊（非高併發設計）

## 💡 常見問題 (FAQ)

**Q: 為什麼部屬後顯示 "Please wait..." 很久？**  
A: Streamlit Cloud 免費版在喚醒時需要 10-30 秒啟動環境，請耐心等待。

**Q: 可以同時多人使用嗎？**  
A: 可以，但每位用戶會看到相同的對決狀態（共享 session）。如需要獨立實例，請各自部署。

**Q: 數據會即時更新嗎？**  
A: 使用 TWSE 日終數據，每天收盤後更新。盤中無法取得即時行情。

**Q: 如何儲存我的對決結果？**  
A: Streamlit Cloud 免費版無持久化儲存，建議截圖保存績效。如需長期儲存，需連接外部資料庫（如 Firebase、Supabase）。

**Q: 可以修改策略程式碼嗎？**  
A: 本版本使用安全的參數化策略。如需完全自定義策略邏輯，請 Fork 原始碼並修改 `StrategyEngine` 類別。
